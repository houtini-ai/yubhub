feat: Add scheduler API endpoints to MCP server

Integrated the new periodic scheduler endpoints into the Yubnub MCP server.

New Tools Added:
- get_feed_schedule: Check auto-run configuration for a feed
  * Shows enabled/disabled status
  * Displays run interval (days)
  * Shows next scheduled run timestamp
  * Shows last run timestamp

- enable_feed_auto_run: Enable automatic periodic runs
  * Configurable interval (1-168 days, default 7 for weekly)
  * Calculates next_run_at based on last_run_at + interval
  * Provides clear feedback on when next run will occur

- disable_feed_auto_run: Disable automatic runs
  * Returns feed to manual-only triggering
  * Clears next_run_at timestamp

API Client Updates:
- Added getFeedSchedule() method
- Added updateFeedSchedule(feedId, enabled, intervalDays) method
- Both methods use new /api/feeds/:id/schedule endpoints

Type Definitions:
- Extended Feed interface with scheduler fields:
  * auto_run_enabled?: number
  * run_interval_days?: number
  * next_run_at?: number | null
- Added FeedSchedule interface for API responses

Tool Descriptions Enhanced:
- Rich context for Claude to understand when to use scheduler tools
- Clear examples of typical intervals (7 days weekly, 14 days bi-weekly)
- Explanation of hourly scheduler worker behavior
- User-friendly messaging about automatic vs manual triggering

Version bumped to 2.1.0 to reflect scheduler feature addition.

All tools tested successfully against staging API endpoints.
